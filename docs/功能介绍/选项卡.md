# 选项卡

因千星的选项卡不支持动态文本，为保证奇匠的创作效率，这里使用了旁门左道实现了动态选项卡，缺点是不同宽度比例的设备，需要在游戏开始前，玩家手动调整UI比例，请参考（我这里没写）如果奇匠确保自己的地图用不上这个功能，就无需让玩家通过这个修改设置。奇匠们可以用移动端体会一下。

1.创建选项卡内容
我们按照同样的步骤，像创建对话文本一样，创建一个结构体，选中选项卡，起名「选项卡」
![示例](/assets/dialogue-function-introduction/创建选项卡变量.png)

2.来到选项卡结构体中，这里只有一个选项卡内容，它的填入内容比较特殊。

![示例](/assets/dialogue-function-introduction/选项卡结构体.png)

3.比如我们有两个选项，`选项一`和`选项二`，我们就用两个\n将它们合并在一起，比如`选项一\n\n选项二`，当有多个选项也同样用\n\n连起来（`1\n\n2\n\n3\n\n4`）

然后填入选项卡的数量，比如两个选项，就填写`2`,最大不能超过`4`，每个选项卡的跳转内容对应对应的`选项卡选择值跳转`，比如选项一就是进入第一个跳转，这个跳转和[跳转对话](./跳转对话)相同

这里我创建了选项一对话和选项二对话，并在选择后跳转过去

![示例](/assets/dialogue-function-introduction/选项卡编辑好.png)
（创建了两个对话变量）

![示例](/assets/dialogue-function-introduction/选项卡后面的变量.png)
（在第一个选项卡选择值跳转填入了选项一对话，选项二同理）

![示例](/assets/dialogue-function-introduction/选项一对话.png)

4.同样添加触发方式
这里同样选择用选项卡触发，选项卡的注意事项请参考[2添加触发方式-站桩对话示例](./创建对话.md#_2-添加触发方式-站桩对话示例)

![示例](/assets/dialogue-function-introduction/创建选项卡节点图.png)
