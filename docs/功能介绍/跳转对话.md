# 跳转对话
你可能会觉得说，正常的对话不是一个对话节点跳另一个对话节点，一条一条播放吗？

但在回看原宝本身的对话时，我发现一个很有意思的地方，就是人物与人物之间它是可能会有时间间隔，并且伴随对话框闪烁一下。但是同一个人对话是没有停顿，并且也不会闪烁

因此我把`一个人+多段对话`做成一个小节点，每个小节点都是内容从上播到下，播完小节点后，`对话列表`切换到下一个小节点播放，直到最后进入跳转对话字段。


![示例](/assets/dialogue-function-introduction/对话列表往下读.png)

而这个对话结构体就是类似大节点，跳转对话的作用是从一个大节点跳转到另一个大节点，通常的作用是切换对话类型（例如站桩->黑幕，站桩->选项卡->站桩）

1.这里有一系列对话类型。在想要跳转的对话类型后面，填上变量名（触发节点图我们后面说），便可以在该对话结束后，跳转到另一个储存对话的变量中。若不填写，则会结束对话（**执行退出对话流操作**）
![示例](/assets/dialogue-function-introduction/跳转对话.png)

2.例如我们想让伊蕾娜的对话结束后，接上一个黑幕文本作为过渡，便创建一个0.5秒的黑幕对话的变量作为过渡（这里不多赘述，请自行查看 [3.为对话添加自定义播放时间-边走边说对话与黑幕文本示例](#_3-为对话添加自定义播放时间-边走边说对话与黑幕文本示例) )

（创建了一个0.5s的黑幕对话用来作过渡）
![示例](/assets/dialogue-function-introduction/创建了一个黑幕过渡.png)

（填入跳转对话中想要跳转的类型后面，比如这里的黑幕）
