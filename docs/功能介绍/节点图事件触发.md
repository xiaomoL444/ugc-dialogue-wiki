# 节点图事件触发

节点图触发，是在播放对话时，执行节点图事件的行为。

每播放一句话，就会触发一次对应序号的事件，若字典里有对应的键值，就会触发对应名字的事件

::: note
在程序中，通常列表第一位序号为0
:::

例如在这里，播放第一句话时会触发序号为`0`名为`触发第一句话事件`的事件，播放第三句话就会触发序号为`2`名为`触发第三句话事件`的事件


::: info
特殊的，当这个人对话完（例如说完第三句话），准备跳转下一个人物的对话时，会额外触发触发一个序号`N+1`的事件，比如这里的`触发结束事件`
:::

::: info
特殊的，当对话结束（即**退出对话流时**，或是跳转对话没内容时，结束对话时），会额外触发一个`***对话结束***`事件（有三个星号）
:::

![示例](/assets/dialogue-function-introduction/节点图触发事件.png)

而在跳转对话中，同样存在`跳转节点图`，原理和上面的`节点图事件触发`一样，在跳转对话时，会`触发节点图事件`

::: warning
若在跳转对话中进入了节点图，会**脱离对话流**，无法执行关闭UI等默认退出对话流事件，需要使用关闭对话节点主动退出对话，或者若只是进行节点图判断，会立刻重新回到对话流，则可以不用关闭对话
:::

![示例](/assets/dialogue-function-introduction/跳转对话的触发节点图.png)

节点图这边，当触发节点图事件时，`对话事件触发时`会收到事件，变量名和事件名顾名思义，需要对他们两个值进行多分支才能获取到正确的事件

::: tip
当奇匠使用`跳转节点`跳转到其他变量对话内容时，事件触发时变量名也是跳转后的变量名
:::
